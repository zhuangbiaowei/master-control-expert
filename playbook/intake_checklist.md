# Universal Intake Checklist (Other)

When information is missing, make best-effort assumptions and label them.

## 1) Objective
- What is the user trying to achieve? (deliverable/decision/publication/implementation)

## 2) Audience
- Who will consume the output? (leaders/devs/students/reviewers/public)

## 3) Constraints
- format (md/doc/ppt/yaml/json)
- language (zh/en)
- word count / depth
- deadline / sequencing
- prohibited terms / style constraints

## 4) Inputs
- existing text/data/links/code
- repo structure
- environment (OS, GPU, versions)

## 5) Preferences
- rigor vs readability
- stability vs performance
- conservative vs aggressive scope

## 6) Acceptance Criteria (“Definition of Done”)
- measurable checks
- examples of “good”
- test cases / review checklist

## 7) Risks / Unknowns
- top failure modes
- what could invalidate results

## 8) Assumptions (explicit)
- fill missing items with reasonable defaults
