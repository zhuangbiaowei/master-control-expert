### Version 1.1 - Philosophy Expert Addition (2024)
**Trigger:** User request "新增一个哲学专家" (Add a philosophy expert)
**Date:** System evolution initiated
**Decision Process:**
1. **Analysis:** Philosophy tasks didn't fit cleanly into existing categories
   - Logic (A) covers formal logic but not ethical reasoning
   - Theory (B) covers frameworks but not philosophical traditions
   - Policy (F) covers compliance but not moral philosophy

2. **Need Assessment:** Philosophy requires specialized handling:
   - Ethical analysis with multiple moral frameworks
   - Conceptual clarification and definition
   - Philosophical argument reconstruction
   - Integration of philosophical traditions

3. **Design Decision:** Create new Category I (Philosophy & Ethics)
   - Follows same pattern as A-H categories
   - Includes specialized prompt skeleton
   - Maintains system consistency

**Changes Made:**
1. Added Category I to prompt catalog
2. Created Philosophy Expert skill specification
3. Updated this evolution protocol

**Validation:**
- New category handles: ethical dilemmas, conceptual analysis, philosophical arguments
- Integrates with existing categories (collaborates with A, B, F)
- Maintains backward compatibility

## Evolution Rules

### Category Addition Protocol
When adding a new category:
1. **Trigger Identification:** Document why existing categories are insufficient
2. **Scope Definition:** Clearly define the new category's domain
3. **Skeleton Design:** Create prompt skeleton following established patterns
4. **Integration Check:** Ensure no conflicts with existing categories
5. **Documentation:** Update all relevant files
6. **Validation:** Test with sample tasks

### Prompt Improvement Protocol
When modifying existing categories:
1. **Issue Identification:** Document specific problems or gaps
2. **Solution Design:** Propose concrete improvements
3. **Impact Assessment:** Evaluate effects on other categories
4. **Implementation:** Update prompt skeletons
5. **Testing:** Validate with representative tasks

### Quality Standards
- All changes must maintain system consistency
- New categories must have clear differentiation from existing ones
- Prompt skeletons must be actionable and specific
- Documentation must be updated synchronously
