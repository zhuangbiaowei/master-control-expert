# Master Control Expert (MCE)

一个用于 Claude Code 的结构化任务处理框架。MCE 将任意请求转化为可执行的、带治理的工作流程。

## 简介

Master Control Expert (MCE) 是一个任务分类和执行框架，帮助 Claude Code 高效处理各种类型的任务。通过标准化的流程和提示模板，MCE 确保每个任务都能得到系统化、高质量的处理。

## 核心流程

```
接收任务 → 需求收集 → 任务分类 → 执行计划 → 交付成果 → 进化评估
```

### 五步工作法

1. **需求收集 (Intake)** - 使用通用检查清单收集必要信息
2. **任务分类 (Classify)** - 将任务归入 A-H 类别
3. **执行控制 (Control)** - 输出控制摘要和目标定义
4. **实际执行 (Execute)** - 应用对应的提示模板并产出具体交付物
5. **进化评估 (Evolve)** - 自我评估，必要时提出类别新增或提示词改进

## 任务分类体系 (A-H)

| 类别 | 名称 | 适用场景 |
|------|------|----------|
| A | 逻辑与论证 | 论证审查、有效性检验、反例分析、"冷水审查" |
| B | 数学/理论解释 | 分层讲解、概念图、学习路径、书单推荐 |
| C | 工程交付 | 系统架构、重构、规范、故障排查、部署 |
| D | 产品/机制设计 | 治理、规则、激励、反滥用、可持续性设计 |
| E | 写作与出版 | 大纲、改写、去重、编辑指导 |
| F | 政策/法律/合规 | 法律推理、合规清单、政策分析 |
| G | 研究整理/学习指南 | 知识图谱、阅读指南、标签系统、文献目录 |
| H | 活动与项目管理 | 会议、论坛、奖项、竞赛、晚宴策划 |

## 使用方法

### 作为 Claude Code 技能使用

在 Claude Code 中激活此技能后，系统会自动遵循 MCE 框架处理任务：

```
/master-control-expert
```

### 手动参考

1. 首先查阅 `playbook/intake_checklist.md` 收集需求
2. 根据任务类型查阅 `playbook/prompt_catalog.md` 中的对应模板
3. 使用 `playbook/control_plan.md` 规划执行
4. 在 `playbook/decision_log.md` 中记录重要决策

## 关键原则

- **直接可操作**：提供清晰的检查清单、表格和编号步骤
- **不拖延**：立即交付，不承诺未来工作
- **明确假设**：当信息不足时，做出合理假设并明确标注
- **追踪决策**：显式记录决策，维护决策日志
- **产出具体交付物**：文件、内容块、可执行步骤
- **自我进化**：持续评估类别覆盖率和提示词效果，主动改进

## 自我进化功能

MCE 具备自我进化能力，能够根据使用情况自动优化：

### 新增类别流程

当任务不属于 A-H 中任何一类时，MCE 会：
1. 评估是否需要新增类别
2. 向用户展示新类别提案（名称、定义、提示词骨架）
3. 获得用户确认后，自动更新相关文件
4. 在 `evolution_protocol.md` 中记录

**触发条件**：
- 任务与 A-H 差异度 > 70%
- 连续 3 个任务落入 Other 且主题相关
- 用户明确要求新增类别

### 提示词改进流程

当某类别的提示词效果不佳时，MCE 会：
1. 诊断问题（遗漏/冗余/顺序不当）
2. 向用户展示改进方案（修改前后对比）
3. 获得用户确认后，更新 `prompt_catalog.md`
4. 升级版本号并记录改进日志

**触发条件**：
- 用户反馈某类别效果不佳
- 发现提示词骨架遗漏关键项
- 用户反复要求覆盖默认部分
- 同一类别连续 2 次需要重大调整

详见 `playbook/evolution_protocol.md`

## 输出示例

对于每个任务，MCE 会产出：

- **控制摘要**：目标、交付物、约束、假设、风险、完成定义
- **执行计划**：里程碑、资源、步骤
- **具体交付物**：实际的文件/内容块
- **决策日志条目**：记录关键决策
